Setting up this blog

Hugo and static websites

I consider myself an old-school html writer: I like to choose a theme and then manually tweak and write my html code directly. I had my hands dirty with Bootstrap when responsive websites started to be a fundamental request, and managed by personal website. This works usually well for small webpages (like the one I had to deal with), both personals and the few conference webpages I had to mantained. It works well but the write-directily-html-code can be sometimes a bit painful. Granted, IDE like VSCode or my (now up to date and running) NeoVim set up makes the process much faster and streamlined: autocomplete html tag and other LSP-capabilities drastically improved the setup. Moreover sometimes for local university servers and special setups, dealing directly with html code becomes the only solutions.

Recently I started reading about Static Site generators: the idea to be able to write readable text and minimal code in Markdown sintax and having a program create the webpage for me sounded very attractive. I was a bit weary of the technical inital step to setup the whole project and still worried that it might become more time comsuming than my previous workflow, but I started to give it a try for a couple of projects.

Advantages and the joy of Markdown

The inital set up was quite easy: install a Static site generator program locally, in my case Hugo, via homebrew and choose the theme in the expanding gallery of themes (of course one can also create his own theme but that will be another project for another time). A more in depth set-up guide is available in the Hugo webiste.

To create your local folder for your webiste it is enouhg to run

hugo new site my-new-site
cd my-new-site
git init
git submodule add github-repo-of-the-theme themes/chosen_theme
echo "theme = 'chosen_theme'" >> hugo.toml

Most of the setting of the website are contained in the file hugo.toml. This file is written in TOML which is a file format for config files. It is usually quite readable and easy to modify (at least for small changes - big changes are more challenging). Once you set up your config file, to add new content (in this case blog posts) one can run

hugo new content content/posts/my-first-post.md

which creates a basic Markdown file called my-first-post usually under the content/posts/ folder. And then one can just write Markdown code. Which means easy text, well-formatted and basic code blocks providing low-level html code without the verbosity of the html tags. A very nice way to create, modify and finalize content!

Preview and HTML creation

This is nice and everything but two important things are missing: how can one preview the actual html page that Hugo will generate? How can one create the actual webpage files that will be uploaded remotely to have a functioning website? Hugo has easy answers to both.

In order to preview the webpages is enough to run the command

hugo server -D

(in the same folder where the Hugo project lives): this will start Hugoâ€™s development server and allows the viewing of the site. In fact once opened in a browser, at the address http://localhost:1313/blog/posts/ (in my setting) there will be a live version of the website that dynamically updates at every save of a relevant file. The -D flag tells hugo to create and display also webpages in the draft form (usually new blog posts are created with the field draft: true and hence not showed by a simple hugo server

Once the website and webpages are in a final form, to actually ask Hugo to “publish” the website is sufficient to run

hugo

When you publish your site, Hugo creates the entire static site in the public directory in the root of your project. This includes the HTML files, and all the other files needed, such as images, CSS files, and JavaScript files. In particular in the folder public one will find all the relevant files that can be copied on a server to display the webpage. Easy and stress free.

Some final thoughts

This showed how to set up and run a static website using Hugo. For this blog I’m using the Bear Cub Theme, free and multilingual (to leave the door open to future posts in Italian). It’s simple, easy to set up and to modify (I only made small modification so I cannot say about more fundamental changes). It’s very minimal, especially the way I’m using it but it has everything that I was looking for to set up my first static blog. Up to now I’m enjoying the experience of writing and modify the pages.
