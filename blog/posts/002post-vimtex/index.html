VimTex and LuaSnip

2025-08-19 Â· by Amos

VimTex

The main tool I will use with NeoVim is the ability to write LaTex getting all the advantages of something like VSCode but in the realm of NeoVim. In order to do that the plugin of choice for me is VimTex.

Things that the VimTex plugin allows you to achieve inside NeoVim:

  • continuously compile LaTex source as you type;
  • debug LaTex compiling directy in NeoVim;
  • forward and backward search between .tex and compiled .pdf(in MacOS this requires the use a different PDFviewer, I use Skim);
  • autocompletion using the TreeSitter Neovim plugin
  • ability to use, edit and create snippets via LuaSnip (more about this later)
  • quick and effetcive .bibtex integration

In other words, everything you might use from an IDE (like VSCode) to write and work with .tex files can be achieved inside NeoVim with the VimTex plugin.

Setting Up

The installation of the plugin is quite straightforward once you translate the Vimscript code to Lua code. What I ended up using, by searching, copying and pasting from various source is the following code in my vimtex.lua file

 1return {
 2  'lervag/vimtex',
 3  lazy = false, -- lazy-loading will disable inverse search
 4  config = function()
 5    vim.g.vimtex_view_method = 'skim'
 6    vim.g.vimtex_compiler_latexmk = {
 7      aux_dir = './.latexmk/aux',
 8      out_dir = '.',
 9    }
10    -- only open quickfix when there are *errors*
11    vim.g.vimtex_quickfix_open_on_warning = 0
12    vim.g.Tex_DefaultTargetFormat = 'pdf'
13    vim.g.Tex_CompileRule_pdf = 'pdflatex'
14  end,
15  keys = {
16    { '<localLeader>l', '', desc = '+vimtex' },
17    { '<localLeader>lb', ':!pdflatex %:p', desc = 'compile pdf' },
18  },
19}

I also included in my init.lua file the following code that tells NeoVim to treat latex differently (for some specific vim options) and to enable LuaSnip by loading the LuaSnip folder and enable autosnippets.

 1-- latex is special
 2vim.api.nvim_create_autocmd({ 'FileType' }, {
 3  pattern = { 'bib', 'tex' },
 4  callback = function()
 5    vim.opt_local.conceallevel = 0
 6    vim.opt_local.wrap = true
 7    vim.bo.shiftwidth = 2
 8    vim.bo.tabstop = 2
 9  end,
10})
11--
12-- Load snippets from ~/.config/nvim/LuaSnip/
13require('luasnip.loaders.from_lua').load { paths = '~/.config/nvim/LuaSnip/' }
14require('luasnip').setup { enable_autosnippets = true }

This makes working with latex files as fast and straightforward as using an IDE.

LuaSnip

The second amazing feature of VimTex is the ability to directly control snippets with Luasnip. This plugin allows to use, create and edit snippets while working with anytype of language. My personal use has been mainly with LaTeX but I will probably give it a try with HTML and CSS if I keep editing my webpages by end.

The installation of LuaSnip is even simpler than VimTeX since you can just copy the .lua code from the GitHub Repo and have it run immediatly via Lazy.

This time around I got some issues while setting up LuaSnip, most likely because I tried to paste several useful config that were creating conflicts in my NeoVim setup. I finally manage to have it work and my luasnip.lua now looks like this:

 1return {
 2  'L3MON4D3/LuaSnip',
 3  version = 'v2.*',
 4  build = 'make install_jsregexp',
 5  event = 'InsertEnter',
 6  config = function()
 7    require('luasnip.loaders.from_lua').load { paths = '~/config/nvim/LuaSnip/' }
 8    local ls = require 'luasnip'
 9    ls.setup {
10      update_events = { 'TextChanged', 'TextChangedI' },
11      enable_autosnippets = true,
12      store_selection_keys = '<Tab>',
13    }
14    -- LuaSnip Keymaps
15    --
16    -- Expand snippets in insert mode with Tab
17    -- Jump forward in through tabstops in insert and visual mode with Control-f
18    --local ls = require 'luasnip'
19    --
20    vim.keymap.set({ 'i' }, '<Tab>', function()
21      ls.expand()
22    end, { silent = true })
23    vim.keymap.set({ 'i', 's' }, '<C-L>', function()
24      ls.jump(1)
25    end, { silent = true })
26    vim.keymap.set({ 'i', 's' }, '<C-J>', function()
27      ls.jump(-1)
28    end, { silent = true })
29
30    vim.keymap.set({ 'i', 's' }, '<C-E>', function()
31      if ls.choice_active() then
32        ls.change_choice(1)
33      end
34    end, { silent = true, desc = 'select autocomplete' })
35  end,
36}

Here my keymap to expand the snippet is set to Tab and I can cycle to the various snippets with the keymaps above. Now whenever I start typing in LaTeX (or Markdown any other language where the snippets are used) and I start type, e.g., se I see the snippet for \section that, once I pressed Tab, automatically expands to \section{POS} and puts the cursor under POS to let me type the title of the section. You can immediately see how this is useful.

But VimTeX has already many native shortcuts to manipulate text and code, like expanding NeoVim capabilities to be more effective with LaTeX. As an example the cse code lets you Change Surrounding Environment: that is, imagine you’re working with and equation and realize it needs to be an align* block instead: with cse align* you’ll get the new Environment with a fantastic speed. Seems like flying!

Moreover LuaSnip allows you to write your own snippets into a file that gets loaded with NeoVim. The setup is quite sel explanatory but the begininng of my file looks like this:

 1--- Abbreviations used in this article and the LuaSnip docs
 2local ls = require 'luasnip'
 3local s = ls.snippet
 4local sn = ls.snippet_node
 5local t = ls.text_node
 6local i = ls.insert_node
 7local f = ls.function_node
 8local d = ls.dynamic_node
 9local fmt = require('luasnip.extras.fmt').fmt
10local fmta = require('luasnip.extras.fmt').fmta
11local rep = require('luasnip.extras').rep
